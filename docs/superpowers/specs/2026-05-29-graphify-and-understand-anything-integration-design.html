Graphify + Understand-Anything

Install both repos as Claude Code skills/plugins, run them against family-travels, and write a short side-by-side note so we know which to reach for next time.

  • StatusApproved · Variant A
  • OwnerElizabeth
  • Co-authorClaude
  • Repofamily-travels

01Goal

Install Graphify and Understand-Anything as Claude Code skills/plugins, run both against the family-travels repo, and produce a brief side-by-side note on what each surfaces best.

Important framing: both tools transform a codebase into an interactive knowledge graph for developer navigation. They are dev-time aids only — neither is a runtime dependency of the travel site itself.

02Non-goals

  • No new runtime feature in the travel site (no graph view inside the album/map UI).
  • No CI integration, no pre-commit hook to regenerate graphs.
  • No Neo4j export, Obsidian vault wiring, or other extras either tool ships.
  • No per-tool config tuning (LLM backend, model overrides) on first pass — accept defaults; revisit only if results disappoint.
  • No commitment to keep both long-term. The comparison note exists precisely to inform a future “which one stays” decision.

03Constraints & context

  • Small static site (Vite + vanilla JS + some React for the 3D gallery). Codebase fits comfortably in either tool's scope.
  • Owner is already running Claude Code with several plugins installed. Plugin-based install is the smoothest path.
  • Owner does not want generated artifacts tracked in git — they go stale fast and bloat the repo.

04Plan Variant A

Install → gitignore → run both → compare, all in one pass.

1

Verify prerequisites

Confirm Python 3.10+ and that /plugin marketplace add works for an arbitrary GitHub source. If either check fails, surface the error rather than working around it.

2

Install Graphify

pip install graphifyy && graphify install

Writes ~/.claude/skills/graphify/SKILL.md and makes /graphify available. User-scoped; no repo changes.

3

Install Understand-Anything

/plugin marketplace add Lum1104/Understand-Anything
/plugin install understand-anything

Enables /understand, /understand-dashboard, /understand-chat, and /understand-diff. User-scoped.

4

Update .gitignore

graphify-out/
.understand-anything/

The only change to the repo's source files.

5

Run Graphify

Invoke /graphify . at the repo root. Skim graphify-out/graph.html and GRAPH_REPORT.md.

6

Run Understand-Anything

Invoke /understand. Skim .understand-anything/knowledge-graph.json and, if useful, open the dashboard via /understand-dashboard.

7

Write the comparison note

Save to docs/superpowers/notes/graphify-vs-understand-anything.md (~150–250 words). Cover: what each tool surfaced, where each was more useful for this repo specifically, which to use when, and any rough edges hit during install or run. Commit this file.

05What gets committed

Artifact Committed?
.gitignore additions Yes
This design spec (md + html) Yes
docs/superpowers/notes/graphify-vs-understand-anything.md Yes — authored prose
graphify-out/ No — gitignored
.understand-anything/ No — gitignored
Installed skills/plugins (in ~/.claude/) N/A — lives in user's home

06Error handling

  • If pip install graphifyy fails on Python version: stop, report the version found, do not auto-upgrade Python.
  • If /plugin marketplace add Lum1104/Understand-Anything fails: report the error verbatim; no workaround retries.
  • If a tool runs but produces empty/broken output: capture the error in the comparison note rather than masking it.
  • If LLM rate or token limits get hit mid-run: stop, note it, let the owner decide whether to retry.

07Success criteria

01 · Skills live Both /graphify and /understand are invokable from this Claude Code session.
02 · Repo clean .gitignore excludes both tools' output directories.
03 · Tools work Both have produced non-empty output for family-travels at least once.
04 · Decision-ready The comparison note exists, is committed, and answers “which to use when” concretely for this repo.
